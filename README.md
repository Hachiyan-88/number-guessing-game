number_guessing_game

このリポジトリは、Java の学習の一環として作成した 数字当てゲーム（Number Guessing Game） です。
コンソール上で動作し、CPU の強さ選択・勝敗判定・エクストラゲーム・再戦機能など、複数の Java の基礎要素を組み合わせて構成されています。

📁 プロジェクト概要

src/numberGuessingGame/ 配下にゲーム全体を構成するクラス群がまとまっています。
CPU と交互に数字を予想し、正解を当てた方が勝利となるシンプルなゲームですが、CPU ロジックやエクストラモードにより戦略的要素があります。

🎮 機能一覧

CPU 対戦（プレイヤー vs CPU）
1～100 の範囲で数字を予想して対戦

CPU のレベルは以下から選択可能
1: 弱い（ランダム予想）
2: 強い（簡易アルゴリズムによる戦略予想）

先攻 / 後攻
通常ゲーム開始時は先攻固定。
エクストラゲームでは、先攻/後攻もランダム決定

勝敗判定
正解の数字を予想したら勝利

ヒント表示：
小さい数字 → 「もっと大きい」
大きい数字 → 「もっと小さい」

エクストラゲーム
発生条件：強いCPUを選択した状態でゾロ目の数字で勝利した場合に発生
概要：　　CPU最強の挑戦者との特別モード
　　　　　クリアターン数を計測

再戦機能
ゲーム終了後、ReplayManager が y/n 入力を処理
エクストラゲームは再戦不可
nextLine() + parseInt() で入力バグ（nextInt の改行問題）を解消済み

🧩 クラス構成

クラス名	役割
NumberBattleGame	メインループ、ゲームの流れ管理
GameLauncher	ゲーム起動・CPUレベル選択・プレイヤー名入力
HumanPlayer	プレイヤーの入力処理・数字予想
CpuPlayer	CPU の共通処理（抽象クラス）
CpuEasy	弱い CPU（ランダム予想）
CpuSmart	強い CPU（範囲絞りアルゴリズム）
extraGame.ExtraGame	エクストラゲームの進行管理
extraGame.CpuSuper	エクストラゲーム専用 CPU ロジック
game.Judge	勝敗判定・ヒント生成
game.ReplayManager	再戦処理（y/n 入力）
display.GameDisplay	ゲーム画面の表示・演出
config.Config	ゲーム設定管理（CPUレベル・プレイヤー名）
main.Main	エントリーポイント（public static void main）
